TODO: Wallet tracking loses Ethereum intra-block ordering. Moralis cache has the correct order for same-block transactions using block_number and transaction_index, but once transactions are converted into LedgerEvents that metadata is dropped. Later rebuild stages sort only by timestamp and then lexicographically by hash/origin, so same-block events can be replayed in the wrong order. In this case the ETH-T -> ETH swap is processed before the prior tx that credits 7 ETH-T, which creates a false insufficient-balance failure in wallet tracking. https://etherscan.io/tx/0x9795e90368cf6e4edacf33e06d455cb94cc4983f10ea0b09e050a899adf82488 and https://etherscan.io/tx/0x41b99a85b5b52e1ef62ab9de87a2a74fb5f779175a0e0eb0165b8dc6904b6156
